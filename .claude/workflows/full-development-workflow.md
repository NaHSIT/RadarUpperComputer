# 完整开发工作流程

## 流程概述

本工作流程定义了从需求到交付的完整闭环，确保每次工作都调用对应的 agent 并完成所有必要步骤。

## 流程图

```
用户需求
    ↓
```

## 详细步骤

### 步骤 1：接收需求

**调用 Agent**：报告 Agent

**任务**：
1. 记录用户需求到工作报告
2. 分配报告编号
3. 创建初始工作报告文件

**输出**：
- `reports/YYYY-MM-DD-任务名称.md`（初始版本）

**检查点**：
- [ ] 需求已记录
- [ ] 报告编号已分配
- [ ] 工作报告文件已创建

---

### 步骤 2：分析需求

**调用 Agent**：智能协调 Agent

**任务**：
1. 阅读相关设计文档（DESIGN.MD）
2. 分析需求的技术影响
3. 识别隐含需求和约束
4. 确定需求优先级

**输出**：
- 需求分析报告
- 影响范围评估

**检查点**：
- [ ] 设计文档已阅读
- [ ] 技术影响已分析
- [ ] 优先级已确定

---

### 步骤 3：拆解任务

**调用 Agent**：需求拆解 Agent

**任务**：
1. 把大需求拆成可执行小任务
2. 定义任务边界和职责
3. 明确输入输出
4. 定义验收标准
5. 排列执行顺序

**输出**：
- 任务清单（TaskCreate）
- 任务依赖关系图
- 每个任务的验收标准

**检查点**：
- [ ] 任务已拆解
- [ ] 依赖关系已明确
- [ ] 验收标准已定义

---

### 步骤 4：分配任务

**调用 Agent**：总指挥 Agent

**任务**：
1. 根据任务类型分配给合适的 Agent
2. 考虑 Agent 的专业领域
3. 确保任务粒度适中
4. 输出清晰的任务描述

**输出**：
- 任务分配单
- 执行计划

**检查点**：
- [ ] 任务已分配
- [ ] 执行计划已制定
- [ ] 资源已确认

---

### 步骤 5：执行任务

**调用 Agent**：开发 Agent（根据任务类型）

**任务类型与 Agent 对应**：

| 任务类型 | 负责 Agent |
|---------|-----------|
| UI 页面/组件 | 前端实现 Agent |
| API 接口/服务 | 后端实现 Agent |
| 协议解析/设备接入 | 协议/设备接入 Agent |
| 算法/质量控制 | 算法/质量控制 Agent |
| 测试数据 | 数据样本/仿真 Agent |

**执行流程**：
1. 接收任务分配
2. 理解任务需求
3. 确认验收标准
4. 实现代码
5. 编写测试
6. 提交代码审核

**输出**：
- 源代码文件
- 单元测试
- 任务完成通知

**检查点**：
- [ ] 代码已实现
- [ ] 测试已编写
- [ ] 代码已提交审核

---

### 步骤 6：代码审核

**调用 Agent**：代码审核 Agent

**任务**：
1. 检查代码质量
2. 检查命名规范
3. 检查错误处理
4. 检查性能问题
5. 生成审核报告

**输出**：
- 代码审核报告
- 问题清单

**检查点**：
- [ ] 代码质量检查完成
- [ ] 问题已记录
- [ ] 审核报告已生成

---

### 步骤 7：架构审查

**调用 Agent**：架构守门 Agent

**任务**：
1. 检查模块边界
2. 检查依赖方向
3. 检查分层架构
4. 检查共享核心库纯净性

**输出**：
- 架构审查报告
- 合规性检查结果

**检查点**：
- [ ] 架构合规性检查完成
- [ ] 依赖方向正确
- [ ] 模块边界清晰

---

### 步骤 8：安全审计

**调用 Agent**：安全审计 Agent

**任务**：
1. 检查输入验证
2. 检查权限校验
3. 检查会话安全
4. 检查数据安全

**输出**：
- 安全审计报告
- 风险清单

**检查点**：
- [ ] 安全检查完成
- [ ] 风险已识别
- [ ] 审计报告已生成

---

### 步骤 9：测试

**调用 Agent**：测试 Agent

**任务**：
1. 编写单元测试
2. 执行单元测试
3. 编写集成测试
4. 执行集成测试
5. 生成测试报告

**输出**：
- 测试用例
- 测试报告
- 覆盖率报告

**检查点**：
- [ ] 单元测试通过
- [ ] 集成测试通过
- [ ] 测试报告已生成

---

### 步骤 10：集成提交

**调用 Agent**：集成/提交 Agent

**任务**：
1. 检查代码审核状态
2. 检查架构审查状态
3. 检查测试通过状态
4. 合并代码到主分支
5. 提交到 git

**输出**：
- 合并报告
- git 提交记录

**检查点**：
- [ ] 所有审核通过
- [ ] 测试通过
- [ ] 代码已合并
- [ ] 代码已提交

---

### 步骤 11：更新文档

**调用 Agent**：文档 Agent

**任务**：
1. 更新设计文档
2. 更新接口文档
3. 更新用户手册
4. 更新 API 文档

**输出**：
- 更新后的文档文件

**检查点**：
- [ ] 设计文档已更新
- [ ] 接口文档已更新
- [ ] 文档与代码一致

---

### 步骤 12：生成工作报告

**调用 Agent**：工作报告 Agent

**任务**：
1. 收集工作信息
2. 统计完成情况
3. 记录交付物
4. 记录问题和风险
5. 规划后续计划
6. 生成完整报告

**输出**：
- 完整的工作报告
- 报告索引更新

**检查点**：
- [ ] 工作报告已生成
- [ ] 报告内容完整
- [ ] 报告索引已更新

---

### 步骤 13：汇报成果

**调用 Agent**：报告 Agent

**任务**：
1. 汇总工作成果
2. 生成汇报内容
3. 向用户汇报
4. 收集用户反馈

**输出**：
- 工作汇报
- 用户确认

**检查点**：
- [ ] 成果已汇总
- [ ] 汇报已完成
- [ ] 用户已确认

## 异常处理

### 1. 代码审核不通过
```
代码审核 Agent 发现问题
    ↓
打回给开发 Agent
    ↓
开发 Agent 修复问题
    ↓
重新提交审核
    ↓
审核通过后继续
```

### 2. 架构不合规
```
架构守门 Agent 发现违规
    ↓
打回给开发 Agent
    ↓
开发 Agent 重构代码
    ↓
重新提交审查
    ↓
审查通过后继续
```

### 3. 测试不通过
```
测试 Agent 发现测试失败
    ↓
打回给开发 Agent
    ↓
开发 Agent 修复 Bug
    ↓
重新执行测试
    ↓
测试通过后继续
```

### 4. 用户反馈问题
```
用户反馈问题
    ↓
报告 Agent 记录问题
    ↓
智能协调 Agent 分析问题
    ↓
分配给对应 Agent 处理
    ↓
处理完成后重新审核
    ↓
确认问题已解决
```

## 流程优化

### 1. 并行执行
- 代码审核、架构审查、安全审计可以并行执行
- 不同模块的开发可以并行执行

### 2. 自动化
- 测试可以自动化执行
- 代码审核可以自动化检查
- 报告可以自动生成

### 3. 持续集成
- 每次提交都触发 CI/CD
- 自动运行测试
- 自动部署到测试环境
