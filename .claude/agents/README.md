# 测风雷达上位机项目 - 多代理协作系统

## 代理编制

### 核心协调层
1. **智能协调 Agent** (`smart-coordinator.md`) - 实时阅读文档、自动调用 agent、智能任务分解
2. **总指挥 Agent** (`coordinator.md`) - 全局进度、任务分配、优先级、阻塞处理
3. **报告 Agent** (`reporter.md`) - 向用户汇报进度、状态、问题

### 质量门禁层
3. **架构守门 Agent** (`architect.md`) - 模块边界、依赖方向、分层合规
4. **代码审核 Agent** (`code-reviewer.md`) - 静态审核 + 系统审核
5. **安全审计 Agent** (`security.md`) - 权限、会话、越权风险

### 需求与设计层
6. **需求拆解 Agent** (`requirements.md`) - 大需求拆成可执行任务

### 实现层
7. **前端实现 Agent** (`frontend-client.md`) - 客户端 UI 实现
8. **前端实现 Agent** (`frontend-radar.md`) - 雷达端 Web UI 实现
9. **后端实现 Agent** (`backend.md`) - 服务层、接口、权限、OTA
10. **协议/设备接入 Agent** (`protocol.md`) - 协议解析、Modbus、TCP/UDP
11. **算法/质量控制 Agent** (`algorithm.md`) - 风场计算、质量评估

### 质量保障层
12. **测试 Agent** (`testing.md`) - 单元测试、集成测试、回归测试

### 集成与发布层
13. **集成/提交 Agent** (`integration.md`) - 代码合并、提交主仓库
14. **发布/版本 Agent** (`release.md`) - 版本号、构建、打包

### 支撑层
15. **文档 Agent** (`docs.md`) - 设计文档、接口文档、用户手册
16. **数据样本/仿真 Agent** (`simulation.md`) - 测试数据、回放文件
17. **工作报告 Agent** (`report-writer.md`) - 生成和管理工作报告

## 工作流程

```
用户需求
    ↓
[报告 Agent] 接收需求，汇报给用户
    ↓
[总指挥 Agent] 分析需求，拆解任务
    ↓
[需求拆解 Agent] 输出可执行任务清单
    ↓
┌─────────────────────────────────────────┐
│         并行执行（按依赖关系）              │
├─────────────────────────────────────────┤
│ [前端实现] [后端实现] [协议] [算法]        │
│     ↓         ↓        ↓      ↓        │
│ [代码审核] ← [架构守门] ← [安全审计]     │
│     ↓                                      │
│ [测试 Agent] 执行测试                      │
│     ↓                                      │
│ [集成 Agent] 合并代码                       │
│     ↓                                      │
│ [发布 Agent] 版本打包                       │
│     ↓                                      │
│ [文档 Agent] 更新文档                       │
└─────────────────────────────────────────┘
    ↓
[报告 Agent] 汇报完成状态给用户
```

## 协作规则

### 1. 权限控制
- 只有 **集成 Agent** 可以提交代码到主仓库
- **代码审核 Agent** 和 **架构守门 Agent** 有否决权
- **总指挥 Agent** 有最终决策权

### 2. 流程门禁
```
开发 Agent 写代码
    ↓
代码审核 Agent 先审（静态 + 系统）
    ↓
架构守门 Agent 再看（分层、依赖）
    ↓
测试 Agent 跑测试
    ↓
集成 Agent 合并到主线并提交
    ↓
发布 Agent 负责版本和打包
```

### 3. 通信协议
- 所有 Agent 通过 **总指挥 Agent** 协调
- **报告 Agent** 负责与用户沟通
- 紧急问题直接上报总指挥

### 4. 质量标准
- 代码审核不通过 → 打回开发 Agent
- 架构不合规 → 打回并通知总指挥
- 测试不通过 → 打回开发 Agent
- 所有打回必须附带具体修复建议

## 快速启动

### 场景 1：新功能开发
```
用户：实现总览页面的风速趋势图
    ↓
报告 Agent：收到，正在分析需求...
    ↓
总指挥 Agent：拆解任务
    ↓
需求拆解 Agent：
  - 任务1：创建 WindTrendChart 组件（前端）
  - 任务2：实现 /api/device/runtime 接口（后端）
  - 任务3：对接 WebSocket wind.realtime 事件（协议）
  - 任务4：定义 WindProfile 数据模型（领域）
    ↓
并行执行各任务
    ↓
代码审核 + 架构审查 + 测试
    ↓
集成提交
    ↓
报告 Agent：完成，风速趋势图已实现
```

### 场景 2：Bug 修复
```
用户：风场数据显示乱码
    ↓
报告 Agent：收到，正在排查...
    ↓
总指挥 Agent：启动紧急修复流程
    ↓
协议 Agent：检查帧解析逻辑
    ↓
代码审核 Agent：审查编码处理
    ↓
修复 + 测试
    ↓
集成提交
    ↓
报告 Agent：已修复，UTF-8 编码问题
```

### 场景 3：代码审查
```
用户：审核 protocolparser.cpp
    ↓
报告 Agent：收到，正在审核...
    ↓
代码审核 Agent：执行静态审核
    ↓
架构守门 Agent：检查依赖合规
    ↓
安全审计 Agent：检查安全风险
    ↓
报告 Agent：审核完成，发现 3 个问题...
```
