# 工作报告目录

本目录存储项目的所有工作报告，包括任务完成报告、每日工作报告、周工作报告等。

## 目录结构

```
reports/
├── README.md                    # 本文件，报告索引
├── YYYY-MM-DD-任务名称.md       # 任务完成报告
├── YYYY-MM-DD-daily.md          # 每日工作报告
├── YYYY-MM-DD-weekly.md         # 周工作报告
└── YYYY-MM-milestone.md         # 里程碑报告
```

## 报告命名规范

### 任务完成报告
```
YYYY-MM-DD-任务名称.md

示例：
2026-07-09-dashboard-page.md
2026-07-10-protocol-parser.md
2026-07-11-websocket-integration.md
```

### 每日工作报告
```
YYYY-MM-DD-daily.md

示例：
2026-07-09-daily.md
2026-07-10-daily.md
```

### 周工作报告
```
YYYY-WXX-weekly.md

示例：
2026-W28-weekly.md  (第28周)
2026-W29-weekly.md  (第29周)
```

### 里程碑报告
```
YYYY-MM-里程碑名称.md

示例：
2026-07-phase1-completion.md
2026-08-phase2-completion.md
```

## 使用说明

### 1. 生成报告
使用工作报告 Agent（report-writer）自动生成报告：
- 任务完成后自动生成任务完成报告
- 每天工作结束后生成每日工作报告
- 每周工作结束后生成周工作报告

### 2. 查询报告
```bash
# 列出所有报告
ls reports/

# 查看特定报告
cat reports/2026-07-09-dashboard-page.md

# 按日期筛选
ls reports/2026-07-*
```

### 3. 报告归档
- 定期整理报告目录
- 删除过期报告
- 归档重要报告
