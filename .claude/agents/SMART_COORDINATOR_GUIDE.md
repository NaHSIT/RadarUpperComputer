# 智能协调 Agent 使用指南

## 概述

智能协调 Agent 是整个多代理协作系统的智能中枢，能够实时阅读开发文档，理解项目需求，并自动调用其他 agent 完成工作。

## 核心能力

### 1. 文档智能阅读
- 实时读取 DESIGN.MD 和其他设计文档
- 理解文档结构和内容
- 提取关键信息和需求
- 识别任务优先级

### 2. 智能任务分解
- 根据文档自动生成任务清单
- 识别任务依赖关系
- 分配给合适的 agent
- 跟踪任务进度

### 3. 自动 agent 调用
- 根据任务类型自动调用对应 agent
- 协调多个 agent 并行工作
- 处理 agent 间的消息传递
- 解决冲突和阻塞

### 4. 进度实时监控
- 监控各 agent 工作状态
- 识别瓶颈和风险
- 自动调整任务分配
- 实时汇报进度

## 使用示例

### 示例 1：自动实现总览页面

**用户输入：**
```
实现总览页面的风速趋势图
```

**智能协调 Agent 执行流程：**

```
1. 阅读文档
   - 读取 DESIGN.MD
   - 定位第 9.1 节（总览页）
   - 提取风速趋势图需求

2. 分析需求
   - 组件：WindTrendChart
   - 功能：实时风速显示、时间窗口切换
   - 数据源：WebSocket wind.realtime

3. 生成任务清单
   - TASK-001：创建 WindTrendChart 组件（前端）
   - TASK-002：实现 /api/device/runtime 接口（后端）
   - TASK-003：对接 WebSocket wind.realtime 事件（协议）

4. 调用 Agent
   - 调用 frontend-client 执行 TASK-001
   - 调用 backend 执行 TASK-002
   - 调用 protocol 执行 TASK-003

5. 监控进度
   - 跟踪各任务状态
   - 识别风险问题
   - 自动调整资源

6. 汇报结果
   - 生成进度报告
   - 汇报给用户
```

### 示例 2：自动修复 Bug

**用户输入：**
```
风场数据显示乱码
```

**智能协调 Agent 执行流程：**

```
1. 诊断问题
   - 阅读相关代码
   - 分析日志
   - 定位问题：UTF-8 编码错误

2. 生成修复任务
   - TASK-001：检查协议解析编码（协议）
   - TASK-002：检查 UI 显示编码（前端）
   - TASK-003：添加编码测试用例（测试）

3. 调用 Agent
   - 调用 protocol 检查解析逻辑
   - 调用 frontend 检查显示逻辑
   - 调用 testing 添加测试

4. 验证修复
   - 运行测试用例
   - 验证修复效果

5. 汇报结果
   - 说明问题原因
   - 展示修复内容
   - 提供预防建议
```

### 示例 3：自动代码审查

**用户输入：**
```
审核 protocolparser.cpp
```

**智能协调 Agent 执行流程：**

```
1. 阅读代码
   - 读取 protocolparser.cpp
   - 分析代码结构
   - 识别关键逻辑

2. 阅读文档
   - 读取 DESIGN.MD 第 6.3 节
   - 理解协议规范
   - 对照实现

3. 生成审查任务
   - TASK-001：静态代码审核（code-reviewer）
   - TASK-002：架构合规审查（architect）
   - TASK-003：安全风险审查（security）

4. 并行调用 Agent
   - 调用 code-reviewer 执行 TASK-001
   - 调用 architect 执行 TASK-002
   - 调用 security 执行 TASK-003

5. 汇总结果
   - 各 agent 审查结果
   - 综合风险评估
   - 提供修复建议

6. 汇报给用户
   - 审查结论
   - 问题清单
   - 改进建议
```

## 工作模式

### 模式 1：全自动模式

**适用场景：**
- 需求明确，无需确认
- 批量任务执行
- 自动化流程

**使用方式：**
```
用户：实现总览页面、风场页面、波束页面
    ↓
智能协调 Agent 自动执行所有任务
    ↓
用户：收到，所有页面已实现
```

### 模式 2：半自动模式

**适用场景：**
- 需要用户确认关键决策
- 复杂任务需要指导
- 风险较高的操作

**使用方式：**
```
用户：优化数据库查询性能
    ↓
智能协调 Agent：
    1. 分析现有查询
    2. 生成优化方案
    3. 询问用户确认
    ↓
用户：确认方案 A
    ↓
智能协调 Agent 执行优化
    ↓
用户：收到，优化完成
```

### 模式 3：交互模式

**适用场景：**
- 需求不明确，需要讨论
- 探索性开发
- 学习和培训

**使用方式：**
```
用户：我想添加数据导出功能
    ↓
智能协调 Agent：
    1. 阅读文档中的导出相关章节
    2. 分析现有导出功能
    3. 生成需求文档
    4. 询问用户需求细节
    ↓
用户：需要支持 CSV 和 NetCDF 格式
    ↓
智能协调 Agent：
    1. 生成详细任务清单
    2. 询问用户确认
    ↓
用户：确认，开始执行
    ↓
智能协调 Agent 执行任务
```

## 文档阅读策略

### 1. 自动识别相关章节

智能协调 Agent 会根据用户需求自动识别文档中的相关章节：

| 用户需求 | 自动识别章节 |
|---------|-------------|
| 实现页面 | 第 8-9 章（界面设计） |
| 实现接口 | 第 18 章（接口设计） |
| 实现协议 | 第 6 章（通信协议） |
| 实现算法 | 第 7 章（算法链路） |
| 修复 Bug | 相关章节 + 代码文件 |
| 代码审查 | 相关章节 + 代码文件 |

### 2. 智能提取需求

智能协调 Agent 会从文档中自动提取：

- 功能需求
- 非功能需求（性能、安全）
- 约束条件
- 验收标准
- 依赖关系

### 3. 自动生成任务

智能协调 Agent 会根据提取的需求自动生成：

- 任务标题
- 任务描述
- 负责 Agent
- 预估工时
- 依赖关系
- 验收标准

## Agent 调用策略

### 1. 智能 Agent 选择

智能协调 Agent 会根据任务内容自动选择最合适的 Agent：

| 任务类型 | 自动选择 Agent |
|---------|---------------|
| UI 页面/组件 | frontend-client / frontend-radar |
| API 接口/服务 | backend |
| 协议解析/设备接入 | protocol |
| 算法/质量控制 | algorithm |
| 单元测试/集成测试 | testing |
| 架构审查 | architect |
| 代码审核 | code-reviewer |
| 安全审计 | security |
| 文档更新 | docs |
| 测试数据 | simulation |
| 版本发布 | release |
| 代码合并 | integration |

### 2. 并行调用

智能协调 Agent 会识别可并行执行的任务，同时调用多个 Agent：

```
任务清单：
- TASK-001：创建前端组件（无依赖）
- TASK-002：创建后端接口（无依赖）
- TASK-003：创建测试用例（依赖 TASK-001, TASK-002）

并行执行：
- 同时调用 frontend 和 backend 执行 TASK-001 和 TASK-002
- TASK-002 完成后，调用 testing 执行 TASK-003
```

### 3. 依赖处理

智能协调 Agent 会自动处理任务依赖：

```
依赖关系图：
TASK-003（领域模型）
    ↓
TASK-002（后端接口）
    ↓
TASK-001（前端组件）

执行顺序：
1. TASK-003（无依赖，先执行）
2. TASK-002（依赖 TASK-003）
3. TASK-001（依赖 TASK-002）
```

智能协调 Agent 会根据情况自动调整：

- 重新分配任务

- 解决冲突

## 常见问题

### Q1：智能协调 Agent 会自动修改代码吗？

**A：** 会的。智能协调 Agent 会自动调用其他 Agent 执行任务，包括修改代码、创建文件等。但它会：
- 在执行前生成任务计划
- 在执行中监控进度
- 在执行后汇报结果
- 遇到关键决策时询问用户

### Q2：如何停止智能协调 Agent 的执行？

**A：** 你可以在任何时候发送"停止"或"暂停"命令，智能协调 Agent 会：
- 停止调用新的 Agent
- 等待当前执行的任务完成
- 汇报当前进度
- 等待你的下一步指示

### Q3：智能协调 Agent 会出错吗？

**A：** 会的。智能协调 Agent 可能会：
- 误解需求
- 选择错误的 Agent
- 生成不合理的任务

但它会：
- 实时监控执行状态
- 识别错误和风险
- 自动调整策略
- 及时汇报问题

### Q4：如何提高智能协调 Agent 的准确性？

**A：** 你可以：
- 提供明确的需求描述
- 指定相关的文档章节
- 提供示例或参考
- 在关键决策时确认
- 及时反馈问题

## 总结

智能协调 Agent 是一个强大的自动化工具，能够：
- 实时阅读文档，理解项目需求
- 自动分解任务，生成执行计划
- 智能调用 Agent，协调并行执行
- 实时监控进度，识别风险问题
- 自动调整策略，确保任务完成

通过智能协调 Agent，你可以实现"说一句话，整个系统自动工作"的效果！
