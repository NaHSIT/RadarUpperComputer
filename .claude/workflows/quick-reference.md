## 常见问题

### Q1：如何选择正确的 Agent？

**A**：根据任务类型选择：
- UI 开发 → 前端实现 Agent
- 后端开发 → 后端实现 Agent
- 协议对接 → 协议/设备接入 Agent
- 算法实现 → 算法/质量控制 Agent
- 代码审核 → 代码审核 Agent
- 测试 → 测试 Agent

### Q2：流程可以跳过某些步骤吗？

**A**：不建议跳过。每个步骤都有其必要性：
- 代码审核：确保代码质量
- 架构审查：确保架构合规
- 安全审计：确保安全
- 测试：确保功能正确
- 报告：记录完整过程

### Q3：流程执行失败怎么办？

**A**：根据失败原因处理：
- 代码审核失败 → 修复代码后重新审核
- 架构不合规 → 重构代码后重新审查
- 测试失败 → 修复 Bug 后重新测试
- 严重问题 → 上报用户决策

### Q4：如何提高流程效率？

**A**：通过以下方式：
- 并行执行：代码审核、架构审查、安全审计可以并行
- 自动化：测试、代码审核可以自动化
- 缓存：相同检查结果可以缓存
- 增量更新：只更新变化部分
