# 工作报告使用指南

## 概述

本指南说明如何在项目中使用工作报告系统，包括报告的生成、查询和管理。

## 工作流程

### 1. 任务完成后生成报告

```
任务完成
    ↓
调用报告 Agent
    ↓
收集工作信息
    ↓
生成报告内容
    ↓
保存到 reports/ 目录
    ↓
更新报告索引
```

### 2. 每日工作结束生成日报

```
每日工作结束
    ↓
调用报告 Agent
    ↓
汇总当日工作
    ↓
生成每日报告
    ↓
保存到 reports/ 目录
```

### 3. 每周工作结束生成周报

```
每周工作结束
    ↓
调用报告 Agent
    ↓
汇总本周工作
    ↓
生成周报告
    ↓
保存到 reports/ 目录
```

## 使用示例

### 示例 1：任务完成后生成报告

**触发条件**：任务状态变为"已完成"

**执行流程**：
1. 智能协调 Agent 检测到任务完成
2. 调用报告 Agent
3. 报告 Agent 收集任务信息
4. 生成任务完成报告
5. 保存到 `reports/YYYY-MM-DD-任务名称.md`

**示例调用**：
```
智能协调 Agent：TASK-001 已完成，请生成报告
报告 Agent：收到，正在生成报告...
报告 Agent：报告已生成：reports/2026-07-09-metric-card.md
```

### 示例 2：每日工作结束生成日报

**触发条件**：每天工作结束时

**执行流程**：
1. 调用报告 Agent
2. 报告 Agent 汇总当日工作
3. 生成每日工作报告
4. 保存到 `reports/YYYY-MM-DD-daily.md`

**示例调用**：
```
用户：生成今日工作报告
报告 Agent：收到，正在汇总今日工作...
报告 Agent：日报已生成：reports/2026-07-09-daily.md
```

### 示例 3：查询历史报告

**触发条件**：需要查看历史报告

**执行流程**：
1. 列出 reports/ 目录下的文件
2. 根据条件筛选报告
3. 读取并展示报告内容

**示例调用**：
```
用户：查看本周的工作报告
报告 Agent：本周共有 3 份报告：
1. 2026-07-09-dashboard-page.md
2. 2026-07-10-protocol-parser.md
3. 2026-07-11-websocket-integration.md
```

## 报告类型说明

### 1. 任务完成报告

**用途**：记录单个任务的完成情况

**内容**：
- 任务概述
- 完成情况
- 交付物清单
- 技术实现
- 质量保证
- 问题和风险
- 后续计划

**生成时机**：任务状态变为"已完成"时

### 2. 每日工作报告

**用途**：记录每天的工作内容

**内容**：
- 今日工作内容
- 工作成果
- 遇到的问题
- 明日计划
- 工作时间统计

**生成时机**：每天工作结束时

### 3. 周工作报告

**用途**：记录每周的工作内容

**内容**：
- 本周工作汇总
- 完成的任务
- 遇到的问题
- 下周计划
- 工作时间统计

**生成时机**：每周工作结束时

### 4. 里程碑报告

**用途**：记录项目里程碑的完成情况

**内容**：
- 里程碑概述
- 完成的工作
- 达成的目标
- 遇到的问题
- 下一阶段计划

**生成时机**：项目里程碑完成时

## 报告质量管理

### 质量标准

1. **完整性**：所有章节都已填写
2. **准确性**：数据统计准确
3. **及时性**：任务完成后及时生成
4. **规范性**：符合报告模板格式

### 审核流程

1. **自动审核**：报告 Agent 检查格式和完整性
2. **人工审核**：总指挥 Agent 审核内容准确性
3. **修正完善**：根据审核意见修正报告
4. **归档保存**：最终版本归档保存

- [ ] 问题记录完整

## 常见问题

### Q1：什么时候需要生成报告？

**A**：以下情况需要生成报告：
- 任务完成后：生成任务完成报告
- 每天工作结束：生成每日工作报告
- 每周工作结束：生成周工作报告
- 项目里程碑完成：生成里程碑报告

### Q2：报告应该包含哪些内容？

**A**：报告应包含：
- 工作概述
- 完成情况
- 交付物清单
- 技术实现
- 质量保证
- 问题和风险
- 后续计划

### Q3：如何保证报告质量？

**A**：通过以下方式保证质量：
- 使用标准报告模板
- 自动检查格式完整性
- 人工审核内容准确性
- 定期整理和归档

### Q4：报告存放在哪里？

**A**：所有报告统一存放在 `reports/` 目录下，按日期和类型组织。

### Q5：如何查询历史报告？

**A**：可以通过以下方式查询：
- 列出 reports/ 目录下的文件
- 按日期筛选
- 按类型筛选
- 按任务名称搜索
