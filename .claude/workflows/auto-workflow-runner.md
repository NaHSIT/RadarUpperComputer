# 自动工作流程运行器

## 概述

本文件定义了自动工作流程运行器，当有新任务时自动调用对应的 agent 并确保流程闭环。

## 使用方式

### 方式 1：一句话触发

```
用户：实现频谱分析页面
    ↓
自动执行完整工作流程
    ↓
用户：收到，频谱分析页面已完成
```

### 方式 2：指定流程

```
用户：按照完整流程实现 XXX 功能
    ↓
自动执行完整工作流程
    ↓
用户：收到，XXX 功能已完成
```

## 流程触发条件

### 自动触发
- 用户提出新需求
- 用户要求实现功能
- 用户要求修复 Bug
- 用户要求进行重构

### 手动触发
- 用户明确要求执行完整流程
- 用户要求调用特定 Agent

## 流程执行规则

### 规则 1：必须调用对应 Agent
每个步骤都必须调用对应的 Agent，不能跳过。

### 规则 2：必须生成输出物
每个步骤都必须有输出物，不能空转。

### 规则 3：必须通过检查点
每个步骤都必须通过检查点，不能跳过验证。

### 规则 4：必须生成报告
流程完成后必须生成工作报告，记录完整过程。

### 规则 5：必须汇报用户
流程完成后必须向用户汇报成果，获得确认。

## 流程监控

### 进度跟踪
- 使用 TaskCreate/TaskUpdate 跟踪任务状态
- 使用 TaskList 查看整体进度
- 使用 TaskGet 获取任务详情

### 质量检查
- 每个步骤都有检查点
- 检查点必须通过才能继续
- 未通过的检查点需要修复后重新检查

### 异常处理
- 任何步骤失败都需要处理
- 失败后需要重新执行该步骤
- 严重问题需要上报用户

## 流程优化

### 1. 并行执行
- 代码审核、架构审查、安全审计可以并行
- 不同模块的开发可以并行
- 测试可以自动化执行

### 2. 缓存机制
- 相同的检查结果可以缓存
- 避免重复执行相同的检查
- 提高流程执行效率

### 3. 增量更新
- 只更新变化的部分
- 避免全量重新生成
- 提高文档更新效率
